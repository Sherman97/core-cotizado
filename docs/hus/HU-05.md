# HU-05 - Registrar una ubicación

## Objetivo
Asociar una ubicación de riesgo a la cotización y clasificarla como completa o incompleta.

## Dependencias previas
HU-01, HU-04

## Alcance dentro del MVP
Esta historia debe implementarse sin ampliar el alcance funcional del MVP del cotizador de seguros de daños.

## Reglas de negocio aplicables
- Una ubicación puede quedar en estado COMPLETA o INCOMPLETA.
- Las ubicaciones incompletas generan alertas pero no bloquean el flujo.

## Criterios de aceptación
- Debe asociarse a la cotización.
- Debe guardar dirección, ciudad, código postal, construcción, giro y demás datos requeridos.
- Debe poder marcarse como completa o incompleta según la información registrada.
- Las ubicaciones incompletas generan alerta pero no bloquean el flujo.

## Cobertura objetivo de la HU
- Mínimo 80% sobre la lógica crítica implementada.
- Debe existir trazabilidad entre criterios de aceptación, Gherkin y tests.
- Debe cubrirse al menos un caso feliz y un caso de borde o error controlado cuando aplique.

## Gherkin
```gherkin
Feature: Registro de ubicación

  Scenario: Registrar una ubicación completa
    Given que existe una cotización activa
    When el usuario registra una ubicación con todos los datos requeridos
    Then el sistema guarda la ubicación
    And marca su estado de validación como "COMPLETA"

  Scenario: Registrar una ubicación incompleta
    Given que existe una cotización activa
    When el usuario registra una ubicación sin todos los datos requeridos
    Then el sistema guarda la ubicación
    And marca su estado de validación como "INCOMPLETA"
    And genera una alerta asociada a la ubicación

```
