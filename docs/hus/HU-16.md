# HU-16 - Consultar historial de versiones

## Objetivo
Listar versiones previas con estado y fecha para consulta simple en el MVP.

## Dependencias previas
HU-15

## Alcance dentro del MVP
Esta historia debe implementarse sin ampliar el alcance funcional del MVP del cotizador de seguros de daños.

## Reglas de negocio aplicables
- Debe respetarse el flujo y estado definidos por el MVP.

## Criterios de aceptación
- Debe listar versión, estado y fecha.
- Debe permitir consultar una versión específica.

## Cobertura objetivo de la HU
- Mínimo 80% sobre la lógica crítica implementada.
- Debe existir trazabilidad entre criterios de aceptación, Gherkin y tests.
- Debe cubrirse al menos un caso feliz y un caso de borde o error controlado cuando aplique.

## Gherkin
```gherkin
Feature: Historial de versiones

  Scenario: Consultar historial de una cotización
    Given que una cotización tiene versiones previas
    When el usuario consulta el historial
    Then el sistema devuelve las versiones asociadas
    And muestra versión, fecha y estado
```
