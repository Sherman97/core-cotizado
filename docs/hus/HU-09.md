# HU-09 - Configurar coberturas de la cotización

## Objetivo
Seleccionar coberturas y parametrizarlas para incluirlas en el cálculo.

## Dependencias previas
HU-01, HU-08

## Alcance dentro del MVP
Esta historia debe implementarse sin ampliar el alcance funcional del MVP del cotizador de seguros de daños.

## Reglas de negocio aplicables
- Debe respetarse el flujo y estado definidos por el MVP.

## Criterios de aceptación
- Debe permitir seleccionar coberturas.
- Debe permitir registrar límite asegurado y deducible cuando aplique.
- Debe quedar asociado a la cotización.
- Debe permitir actualizar configuración previa.

## Cobertura objetivo de la HU
- Mínimo 80% sobre la lógica crítica implementada.
- Debe existir trazabilidad entre criterios de aceptación, Gherkin y tests.
- Debe cubrirse al menos un caso feliz y un caso de borde o error controlado cuando aplique.

## Fases TDD esperadas
### Red

- Mapear cada escenario Gherkin a pruebas.

### Green

### Refactor

## Gherkin
```gherkin
Feature: Configuración de coberturas

  Scenario: Guardar coberturas seleccionadas
    Given que existe una cotización activa
    And existen coberturas disponibles
    When el usuario selecciona coberturas y define sus parámetros
    Then el sistema guarda las coberturas asociadas a la cotización

  Scenario: Actualizar coberturas previamente registradas
    Given que la cotización ya tiene coberturas configuradas
    When el usuario modifica la selección o parámetros
    Then el sistema actualiza la configuración de coberturas

```
