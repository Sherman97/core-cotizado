# HU-02 - Consultar cotización por folio

## Objetivo
Recuperar una cotización existente para continuar el proceso.

## Dependencias previas
HU-01

## Alcance dentro del MVP
Esta historia debe implementarse sin ampliar el alcance funcional del MVP del cotizador de seguros de daños.

## Reglas de negocio aplicables
- Debe respetarse el flujo y estado definidos por el MVP.

## Criterios de aceptación
- Debe devolver datos básicos de la cotización.
- Debe devolver estado y versión actual.
- Si el folio no existe, debe responder con error controlado.

## Cobertura objetivo de la HU
- Mínimo 80% sobre la lógica crítica implementada.
- Debe existir trazabilidad entre criterios de aceptación, Gherkin y tests.
- Debe cubrirse al menos un caso feliz y un caso de borde o error controlado cuando aplique.

## Fases TDD esperadas
### Red
- Diseñar tests primero.
- Mapear cada escenario Gherkin a pruebas.
- Confirmar fallo inicial.

### Green
- Implementar el mínimo código para pasar los tests.
- No agregar comportamiento fuera de la HU.

### Refactor
- Mejorar diseño interno sin alterar comportamiento observable.
- Mantener pruebas en verde.

## Gherkin
```gherkin
Feature: Consulta de cotización por folio

  Scenario: Consultar una cotización existente
    Given que existe una cotización con un folio válido
    When el usuario consulta la cotización por folio
    Then el sistema retorna la información de la cotización
    And muestra su estado actual
    And muestra su versión actual

  Scenario: Consultar un folio inexistente
    Given que no existe una cotización con el folio consultado
    When el usuario intenta consultar la cotización
    Then el sistema responde que la cotización no fue encontrada

```
