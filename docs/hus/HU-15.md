# HU-15 - Generar una nueva versión de la cotización

## Objetivo
Crear un nuevo snapshot editable enlazado a la versión previa.

## Dependencias previas
HU-01

## Alcance dentro del MVP
Esta historia debe implementarse sin ampliar el alcance funcional del MVP del cotizador de seguros de daños.

## Reglas de negocio aplicables
- Cada versión se maneja como snapshot de negocio enlazado a la anterior.

## Criterios de aceptación
- Debe crearse una nueva versión ligada a la anterior.
- Debe conservar referencia a parent_quote_id.
- La nueva versión debe iniciar editable.
- La versión previa no debe perderse.

## Cobertura objetivo de la HU
- Mínimo 80% sobre la lógica crítica implementada.
- Debe existir trazabilidad entre criterios de aceptación, Gherkin y tests.
- Debe cubrirse al menos un caso feliz y un caso de borde o error controlado cuando aplique.

## Gherkin
```gherkin
Feature: Versionado de cotización

  Scenario: Crear nueva versión de cotización
    Given que existe una cotización previa
    When el usuario solicita generar una nueva versión
    Then el sistema crea una nueva cotización relacionada con la anterior
    And incrementa el número de versión
    And conserva el historial de la versión previa

```
