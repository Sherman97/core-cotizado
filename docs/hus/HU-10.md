# HU-10 - Ejecutar cálculo de la cotización

## Objetivo
Calcular prima por ubicación válida, consolidar prima neta y prima comercial.

## Dependencias previas
HU-03, HU-05, HU-09

## Alcance dentro del MVP
Esta historia debe implementarse sin ampliar el alcance funcional del MVP del cotizador de seguros de daños.

## Reglas de negocio aplicables
- El cálculo se realiza por ubicación y luego se consolida.
- Solo se calculan ubicaciones válidas.

## Criterios de aceptación
- Debe evaluar las ubicaciones registradas.
- Debe calcular solo ubicaciones válidas.
- Las incompletas generan alerta pero no bloquean.
- Debe consolidar resultado total.
- Debe actualizar el estado de la cotización.

## Cobertura objetivo de la HU
- Mínimo 80% sobre la lógica crítica implementada.
- Debe existir trazabilidad entre criterios de aceptación, Gherkin y tests.
- Debe cubrirse al menos un caso feliz y un caso de borde o error controlado cuando aplique.

## Gherkin
```gherkin
Feature: Cálculo de cotización

  Scenario: Calcular una cotización con ubicaciones válidas
    Given que la cotización tiene datos generales válidos
    And tiene ubicaciones válidas registradas
    And tiene coberturas configuradas
    When el usuario ejecuta el cálculo
    Then el sistema calcula la prima por cada ubicación válida
    And consolida la prima neta
    And calcula la prima comercial
    And actualiza el estado de la cotización

  Scenario: Calcular una cotización con ubicaciones incompletas
    Given que la cotización tiene al menos una ubicación válida
    And tiene una o más ubicaciones incompletas
    When el usuario ejecuta el cálculo
    Then el sistema calcula solo las ubicaciones válidas
    And registra alertas para las ubicaciones incompletas
    And entrega resultado parcial calculado

```
